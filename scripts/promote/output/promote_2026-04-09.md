## 이번에 한 일

오늘은 암호화폐 자동매매 시스템의 인프라를 한 단계 업그레이드하는 작업에 집중했습니다.

가장 큰 변화는 **OHLCV 데이터 저장소를 구글 드라이브에서 GitHub 레포로 전환**한 것입니다. 업비트 상위 200개 코인 × 6개 타임프레임 데이터를 수집하는 파이프라인을 구축했고, 이를 GitHub에서 직접 관리할 수 있도록 리팩토링했습니다.

또한 **텔레그램 채널 자동 홍보 스크립트**(`scripts/promote/`)를 새로 추가하고, GitHub Actions `promote` 워크플로우를 연동해 JSONL 파일 없이도 안정적으로 동작하도록 구성했습니다. CI/CD 과정에서 `ANTHROPIC_API_KEY` 시크릿명도 `_01`로 정리했습니다.

매매 로직 측면에서는 **매매법 진입·청산 조건을 개선**하고 관련 페이지 문서도 업데이트했습니다. (2번 주기 진입, 5번 즉시매도, 3→4→4번 청산 흐름 반영)

## 배운 점

> "저장소 선택 하나가 파이프라인 전체 복잡도를 바꾼다."

구글 드라이브 연동은 처음엔 편해 보였지만, GitHub Actions와 통합하는 순간 인증·경로 관리가 상당히 번거로웠습니다. GitHub 레포로 일원화하니 버전 관리, 자동화, 협업 모두 훨씬 깔끔해졌습니다. **"단순한 것이 강하다"** 는 걸 다시 한번 체감한 하루였습니다.
