# AI와 함께한 1일 작업 일지 (2026-04-09)

## 이번에 한 일

오늘은 암호화폐 자동 매매 시스템의 인프라를 대폭 개선하는 작업에 집중했다. 크게 세 가지 축으로 작업이 진행됐다.

**① OHLCV 데이터 수집 안정화**
기존 구글 드라이브 기반의 OHLCV 저장소를 GitHub 레포로 완전히 이전했다. 업비트 상위 200개 종목 × 6개 타임프레임을 수집하는 과정에서 타임아웃 문제가 발생했고, 타임프레임별 수집 기간을 조정하는 방식으로 해결했다. 백테스트(`backtest.py`)도 Parquet 파일을 우선 로드하고 API는 폴백으로만 활용하도록 개선했다.

**② 매매 로직 리팩토링**
매매법 진입·청산 조건을 전면 개선했다. 2번 주기 진입, 5번 즉시매도, 3→4→4번 청산 시퀀스가 새롭게 정립됐고, 변경사항은 문서 페이지에도 즉시 반영했다.

**③ 텔레그램 자동 홍보 파이프라인 구축**
GitHub Actions 기반의 promote 워크플로우를 새로 추가했다. JSONL 파일이 없어도 동작하도록 방어 로직을 넣었고, `ANTHROPIC_API_KEY` 시크릿명도 통일했다. 이제 홍보 콘텐츠 자동 생성부터 텔레그램 채널 발송까지 파이프라인이 완성됐다.

## 주요 성과

- ✅ OHLCV 저장소 GitHub 이전 완료 (구글 드라이브 의존성 제거)
- ✅ 타임아웃·수집 오류 2건 핫픽스
- ✅ 백테스트 로컬 Parquet 우선 로드로 API 비용 절감
- ✅ 텔레그램 자동 홍보 워크플로우 end-to-end 완성

## 배운 점

> "저장소 위치 하나가 파이프라인 전체 안정성을 좌우한다."

구글 드라이브는 인증 만료, 용량 제한, 속도 이슈가 복합적으로 터졌다. GitHub 레포로 옮기자마자 Actions 연동이 훨씬 단순해졌다. 외부 스토리지 의존은 최대한 늦게, 꼭 필요할 때만 도입하는 게 맞다는 걸 다시 한번 확인했다.
