# AI와 함께한 하루 작업 일지 (2026-04-09)

## 이번에 한 일

업비트 200개 티커의 OHLCV 데이터를 자동 수집하는 파이프라인을 완성했다. `scripts/ohlcv/collector.py` (531줄), GitHub Actions 워크플로우, 상태 JSON 파일, 그리고 `ai-trading.html`에 OHLCV 상태 섹션까지 한 번에 구축했다. 매일 KST 00:00에 자동 실행되어 15min → 1H → 4H → 1D → 1W → 1M 순서로 전 타임프레임을 수집하는 구조다.

## 주요 성과

처음엔 구글 드라이브 연동을 목표로 했지만, 서비스 계정의 저장 용량 제한이라는 벽에 부딪혔다. 폴더 생성은 성공했는데 파일 업로드에서 계속 실패 — 원인은 서비스 계정은 Drive 저장 용량이 0이라는 구조적 문제였다. 빠르게 방향을 전환해 **GitHub 레포 직접 저장 방식**으로 리팩토링했고, 드라이브 의존성을 완전히 제거했다. 덕분에 외부 인증 설정 없이도 `data/ohlcv/` 폴더에 parquet 파일이 자동 커밋되는 깔끔한 구조가 완성됐다.

## 배운 점

> "폴더 생성은 되는데 파일 업로드가 안 된다면, 용량 문제를 먼저 의심하라."

구글 드라이브 서비스 계정은 용량을 공유하지 않는다는 점을 실제로 부딪혀보고 나서야 명확히 알게 됐다. Claude와 함께 에러 로그를 바로바로 분석하고 대안을 선택하는 과정이 빨랐던 게 이번 작업의 핵심이었다. 삽질을 줄여주는 건 결국 빠른 원인 파악이다.
